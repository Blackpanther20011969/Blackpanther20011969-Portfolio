Flask · CRUD · API-Design

Flask API CRUD

Bücher-API mit Struktur, Routen und JSON

Projektstruktur

flask_api_project/
├── app.py
├── books.py
├── models.py
└── venv/

Datenmodell

books = [
 { "id": 1, "title": "Der Hobbit", "author": "Tolkien" },
 { "id": 2, "title": "Harry Potter", "author": "Rowling" }
]

Kleine In-Memory-Struktur für einfache Tests ohne Datenbank.

CRUD Endpunkte

GET

/api/books → alle Bücher

POST

/api/books → neues Buch

PUT

/api/books/id → ändern

DELETE

/api/books/id → löschen

Flask Routing

@books_bp.route('/api/books', methods=['GET'])
@books_bp.route('/api/books', methods=['POST'])
@books_bp.route('/api/books/', methods=['PUT'])
@books_bp.route('/api/books/', methods=['DELETE'])

API testen

curl http://localhost:5000/api/books

curl -X POST -H "Content-Type: application/json" \
-d '{"title":"1984"}' \
http://localhost:5000/api/books

Was ich gelernt habe

  • CRUD-Struktur sauber aufbauen
  • HTTP-Methoden gezielt einsetzen
  • Daten mit JSON strukturieren
  • API gezielt testen (Postman / curl)
