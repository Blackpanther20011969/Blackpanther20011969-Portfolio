Übung: Todo-API mit Flask

Diese Übungsseite zeigt eine einfache REST-API für eine Todo-Liste. Der Fokus liegt auf grundlegenden API-Konzepten: Endpunkte definieren, Daten als JSON zurückgeben, neue Einträge erstellen, bestehende Einträge aktualisieren und Fehler verständlich behandeln.

Die Übung ergänzt meine technischen Notizen, weil sie zeigt, wie ich aus einem Konzept eine nachvollziehbare kleine API-Struktur ableite und dokumentiere.

Einordnung:
Die Daten werden bewusst in einer einfachen Liste gespeichert. Das ist für eine Lernübung übersichtlich und zeigt die Grundlogik, ersetzt aber keine produktive Datenbanklösung.

1. Ziel der Übung

Routing

Endpunkte definieren

Die API stellt verschiedene URLs bereit, über die Todos gelesen, angelegt, geändert und gelöscht werden können.

CRUD

Operationen abbilden

Die Todo-Liste zeigt Create, Read, Update und Delete anhand einer leicht verständlichen Ressource.

Fehler

Statuscodes nutzen

Fehlende Daten oder nicht gefundene Einträge werden mit passenden HTTP-Statuscodes zurückgegeben.

2. Geplante API-Endpunkte

Die Todo-API verwendet typische REST-Endpunkte. Jeder Endpunkt steht für eine konkrete Aktion auf der Ressource Todo.

GET /api/todos

Alle Todos abrufen

GET /api/todos/<id>

Ein bestimmtes Todo abrufen

POST /api/todos

Neues Todo erstellen

PUT /api/todos/<id>

Todo aktualisieren

DELETE /api/todos/<id>

Todo löschen

3. Datenmodell

Jedes Todo besitzt einfache Felder. Dadurch bleibt die Übung übersichtlich, zeigt aber trotzdem typische Strukturen einer API.

id

Automatische ID

Eindeutige Kennung für jedes Todo.

title

Pflichtfeld

Kurzer Titel des Todos. Ohne Titel wird kein neues Todo erstellt.

description

Optional

Zusätzliche Beschreibung des Todos.

done

Status

Wert true oder false, um den Bearbeitungsstand zu zeigen.

4. Beispielcode

Der folgende Code zeigt eine einfache Todo-API mit Flask. Die Daten liegen in einer Liste, damit die CRUD-Logik gut nachvollziehbar bleibt.

Python · Flask · Todo-API 
from flask import Flask, jsonify, request

app = Flask(__name__)

todos = [
    {
        "id": 1,
        "title": "Flask API lernen",
        "description": "REST API mit Flask erstellen",
        "done": False
    },
    {
        "id": 2,
        "title": "Dokumentation ergänzen",
        "description": "Endpunkte und Statuscodes beschreiben",
        "done": True
    }
]

next_id = len(todos) + 1


@app.route("/api/todos", methods=["GET"])
def get_todos():
    return jsonify(todos)


@app.route("/api/todos/<int:todo_id>", methods=["GET"])
def get_todo(todo_id):
    todo = next((item for item in todos if item["id"] == todo_id), None)

    if not todo:
        return jsonify({"error": "Todo nicht gefunden"}), 404

    return jsonify(todo)


@app.route("/api/todos", methods=["POST"])
def create_todo():
    global next_id

    if not request.json or "title" not in request.json:
        return jsonify({"error": "Titel ist erforderlich"}), 400

    todo = {
        "id": next_id,
        "title": request.json["title"],
        "description": request.json.get("description", ""),
        "done": request.json.get("done", False)
    }

    todos.append(todo)
    next_id += 1

    return jsonify(todo), 201


@app.route("/api/todos/<int:todo_id>", methods=["PUT"])
def update_todo(todo_id):
    todo = next((item for item in todos if item["id"] == todo_id), None)

    if not todo:
        return jsonify({"error": "Todo nicht gefunden"}), 404

    if not request.json:
        return jsonify({"error": "Daten müssen im JSON-Format sein"}), 400

    todo["title"] = request.json.get("title", todo["title"])
    todo["description"] = request.json.get("description", todo["description"])
    todo["done"] = request.json.get("done", todo["done"])

    return jsonify(todo)


@app.route("/api/todos/<int:todo_id>", methods=["DELETE"])
def delete_todo(todo_id):
    todo = next((item for item in todos if item["id"] == todo_id), None)

    if not todo:
        return jsonify({"error": "Todo nicht gefunden"}), 404

    todos.remove(todo)

    return jsonify({"result": True})


if __name__ == "__main__":
    app.run(debug=True)

5. Erklärung der wichtigsten Bausteine

In-Memory

Einfache Datenhaltung

Die Todos werden in einer Python-Liste gespeichert. Das macht die Übung leicht verständlich.

Route

Eine Route pro Aktion

Jede Route entspricht einer CRUD-Operation und ist über eine klare URL erreichbar.

Validierung

Eingaben prüfen

Beim Erstellen eines Todos wird geprüft, ob ein Titel vorhanden ist.

Statuscodes

Rückmeldungen geben

Die API nutzt z. B. 201 für erstellt, 400 für ungültige Daten und 404 für nicht gefundene Todos.

6. Testen der API

Eine API kann mit Tools wie Postman oder mit einfachen Terminal-Befehlen getestet werden. Wichtig ist, Anfrage, Methode, URL und JSON-Daten sauber zu unterscheiden.

Beispiel · curl 
curl -X POST http://127.0.0.1:5000/api/todos \
  -H "Content-Type: application/json" \
  -d '{"title": "API testen", "description": "POST-Anfrage prüfen"}'
Tipp:
Beim Testen lohnt es sich, bewusst Fehlerfälle auszuprobieren: fehlender Titel, falsche ID oder ungültige JSON-Daten.
