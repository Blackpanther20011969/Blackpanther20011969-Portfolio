Fortgeschrittene API-Konzepte

Flask, Datenbanken, Authentifizierung & mehr

1. Datenbankanbindung mit SQLAlchemy

Flask lässt sich hervorragend mit SQLAlchemy als ORM verbinden. Wir definieren Models, konfigurieren die Datenbank und binden diese in unsere API ein.

pip install flask-sqlalchemy

models.py

// Dein Book-Modell mit SQLAlchemy
from flask_sqlalchemy import SQLAlchemy
from datetime import datetime

db = SQLAlchemy()

class Book(db.Model):
    __tablename__ = 'books'
    id = db.Column(db.Integer, primary_key=True)
    title = db.Column(db.String(200), nullable=False)
    author = db.Column(db.String(100), nullable=False)
    year = db.Column(db.Integer)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)

2. Authentifizierung mit JWT

Wir sichern unsere API durch JSON Web Tokens.

pip install flask-jwt-extended

Beispiel-Routen:

  • /api/register – Benutzer registrieren
  • /api/login – Benutzer einloggen & Token erhalten

3. Validierung mit Marshmallow

Validierung von Nutzerdaten über flask-marshmallow sorgt für saubere Eingabeverarbeitung:

pip install marshmallow flask-marshmallow

4. API-Dokumentation mit Swagger

Mit Flasgger kannst du automatisch interaktive API-Dokumentation erzeugen:

pip install flask-swagger-ui flasgger

Swagger zeigt deine Endpunkte übersichtlich im Browser – inklusive Beispielantworten.

Best Practices

  • Sicherheit: Authentifizierung, HTTPS, Rate Limiting
  • Versionierung: z. B. /api/v1/books
  • Fehlerbehandlung: Einheitliche Fehlerstruktur, klare Statuscodes
  • Dokumentation: Nutze Swagger/OpenAPI!
